## 帖子1：普通人要 OpenClaw 有什么用？

**问题**：普通人要 OpenClaw 有什么用？

### 回答1 [匿名] (609 赞同)

**核心观点**：OpenClaw 本质上是对 PC 的个人 AI 助理。

**关键论述**：

1. **使用前提**
   - 需要用 PC 办公的人才能发挥其价值
   - 不装到主力机就没啥用
   - 它需要访问你的文件、执行电脑上的命令

2. **与 Claude Code 的区别**
   - Claude Code 读文件与执行命令需要确认，OpenClaw 给完权限就直接执行
   - Claude Code 不能用手机访问，OpenClaw 打通了 PC 与手机
   - 可以从手机直接给 PC 发命令

3. **为什么都在吹 macmini**
   - OpenClaw 官方内置的大多数 skill 是基于 mac 的
   - 同步苹果日历等功能专门针对苹果

4. **本地部署的意义**
   - 需要能访问本地所有文件、程序
   - 只有开源软件才能让人放心授权如此大的权限
   - 核心意义是访问本地所有文件，运行本地所有程序

5. **关于 windows/linux**
   - 只要想做的事在 windows/linux 下有 CLI 程序能做到，完全没问题
   - 可以自己定制第三方 skills

6. **OpenClaw vs Claude Code**
   - 主要差别：打通手机 + 免确认
   - 对于程序员来说，CC 能做 OpenClaw 绝大多数事情
   - 除非 CC 专门用于个人助理才适合开免确认

## 帖子2：OpenClaw会不会在国内突然用不了？

**问题**：OpenClaw这种"自己会学新技能"的AI助手，会不会像安卓手机一样——在国内突然用不了？

### 回答 [匿名] (2 赞同)

**核心观点**：不太可能走安卓那条路

**关键论述**：

1. **OpenClaw 本质**
   - 不是平台+封闭核心服务的模式
   - 只是 AI 调度框架，负责把模型、工具、执行流程串起来
   - 本身不垄断任何关键资源

2. **本地部署的优势**
   - 模型可以换，本地模型也能跑
   - 工具链可以自己接
   - 执行环境可以是云服务器，也可以是自己的机器

3. **大厂特供版**
   - 各个大厂都已推出自己的"龙虾"
   - 目的是抢占 AI 流量，让用户和自己的模型算力生态绑定
   - 用他们的版本是因为省事，不是因为没选择

4. **关于企业限制**
   - 企业环境里天然有权限、数据、安全边界的问题
   - 项目诞生几月，有漏洞太正常
   - 全球爆火背景下，黑客盯着开源代码找漏洞
